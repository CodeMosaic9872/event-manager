# Platform modules — backend integration status

Below are the **functional areas where application–backend integration is implemented** (the live app talks to the server for these features).

---

## Completed (integrated)

- **Authentication** — sign-in, registration, OTP, sessions, refresh  
- **User profile** — account details and profile images  
- **Supplier marketplace** — supplier listings, supplier profile pages, search suggestions  
- **Supplier dashboard** — supplier profile setup and updates, service areas, media and gallery uploads  
- **Reviews & favorites** — supplier reviews and saved suppliers  
- **Jobs / tenders** — posting jobs, editing and publishing, applications, supplier job feed (recommended / applied), closing jobs and selecting a supplier  
- **Supplier referrals** — referral link for suppliers  
- **AI planning assistant** — chat conversations  
- **Notification preferences** — user notification settings  
- **Categories & taxonomy** — event types, categories and subcategories (including admin management of taxonomy)  
- **Subscription plans** — public plan catalog for supplier join checkout, payment summary, and admin premium-package editor (read from server)  
- **Admin console** — supplier management (list, approvals, moderation, creation, profiles, export), jobs and applications, users, referrals, dashboard metrics, notifications monitoring, automation and supplier–job matching, AI usage oversight  

---

## Not integrated in the app UI yet (examples)

These may exist elsewhere in the product roadmap or only partially; the main customer-facing app flows above are separate:

- End-to-end **payments / checkout** wiring in every screen  
- **Subscription / billing plans** — admin save/persist from premium-packages editor (read is integrated)  
- **Concept store / concept publishing** flows (beyond static or draft UI)  
- **Admin premium package pricing** persistence (configuration screen may be UI-only until backend is connected)
